- **2026-05-03** — After every substantive turn, the main agent must silently delegate to `Sankofa` subagent and only surface `📓 Journal updated.` when an entry was actually added.

- **2026-05-04** — In Codex chat, a Codex subagent may stand in for VS Code/Copilot `runSubagent(agentName: "Sankofa")`; it must follow Sankofa rules and update only `docs/JOURNAL.md`, not a separate memory or handoff file.
- **2026-05-04** — Club crests must never be recolored, cropped, filtered, or passed through generative models. They render as-is from `/public/clubs/<slug>.svg`.
- **2026-05-04** — The same Challenge card system serves five surfaces (hero spotlight, hero-mobile, `/challenges/[id]` detail, `/challenges` list card, dashboard widget); geometry differs per surface, layers and data are shared.
- **2026-05-04** — Headline copy on Challenge cards is selected deterministically on `challenge.id` from a phrase library indexed by stage + tone + club voice — varied across challenges, stable across re-renders.
- **2026-05-11** — No cross-schema joins between engines; all inter-engine communication flows through the outbox event bus only. Reason: independent scaling, zero tight coupling.
- **2026-05-11** — Trust clearances gate **records** (who can see/access data); Moderation gates **public surfaces** (what gets published). These two must never be confused; Trust is for fine-grained access control, Moderation is for brand safety and user safety.
- **2026-05-12** — Backend: modular monolith pattern with **one Laravel module per engine** (Module/Competition, Module/RpEconomy, Module/Match, etc.). Each module owns its schema; no cross-schema foreign keys. Inter-module comms via event bus + ID references only. Integrity workers enforce eventual consistency.
- **2026-05-12** — Schema isolation: each engine module has its own Postgres schema. Cross-module queries forbidden. Use IDs + outbox pattern + async workers for inter-engine effects (e.g., Competition publishes event → RpEconomy worker consumes → RP ledger updated).
- **2026-05-12** — PHP runtime: **FrankenPHP in worker mode** (preferred for concurrency). Laravel Octane as fallback. No traditional CGI.
- **2026-05-12** — Outbox pattern is **mandatory** for all user-triggered writes that trigger cross-engine effects (e.g., match result → RP ledger + notification). Write to local schema + outbox in same transaction, async worker publishes to Redis Streams.
- **2026-05-12** — **Idempotency required** on every user-triggered write endpoint. Use `Idempotency-Key` header + deduplication store (Redis, 24h TTL). Prevent double-mutations (e.g., duplicate RP transfers, duplicate payment captures).
- **2026-05-12** — **FanBuzz module has hard write-isolation** from RP, Match, and Competition schemas. It reads RP/Match/Competition events but never mutates them. Read-only views only.
- **2026-05-12** — **RP economy**: only mutated via ledger entries in `rp_ledger` table (append-only). No in-place balance updates. Derived balance views materialized from ledger. Partitioned by season for query performance.
- **2026-05-12** — **Monetary amounts**: always stored as **integer minor units** (satoshis/pesewas; never floats). Conversion to display currency happens at API boundary (Twig view layer).
- **2026-05-12** — **Stage 0 is gate**: All 17 module scaffolds, Deptrac+Pest Architecture lints, schema-per-module isolation, event bus + outbox proven end-to-end, analytics envelope, Admin Config registry, Sanctum+OTP+scope auth, observability baseline **must ship before ANY user-facing feature**. Non-negotiable.
- **2026-05-12** — **Football-drama ordering constraint**: Stage 2 (The Match) with Trust V1 and super-admin override must be rock-solid before Stage 5 (Drama Layer) launches. Drama entirely depends on Trust being real. Revenue (Stage 6) is **last major feature before scale-out (Stage 8), never first**.
- **2026-05-12** — **Cross-stage non-deferrables**: Analytics, Admin Config, Trust gate (access control), Moderation gate (brand safety), architecture lint (Deptrac), and Super Admin audit path are required to exist and function in **every stage from Stage 0 onward**. Explicit forbid to defer these or stub them.
- **2026-05-12** — **Archive-never-delete**: All records are immutable after creation. Logical deletion (soft delete, archival flag, or status change) replaces hard deletion. Rationale: audit trail integrity, Trust/Moderation backtracing, financial reconciliation, and dispute resolution all depend on records never vanishing.

## Craftsmanship Standards

- **2026-05-20** — **File size hard limits**: no source file > 400 LOC, no function/method > 50 LOC, no class > 250 LOC. Violation is a refusal trigger for Implementer. Reason: maintainability, testability, code review.
- **2026-05-20** — **Naming discipline**: verbs for functions (`calculateStake()`), nouns for variables/classes; no abbreviations except project-canonical (RP, WP, ADR, OTP, TTL, DAU, KPI); booleans start with `is`/`has`/`can`/`should`; internal keys are `snake_case` strings (`result_confirmed`), display strings are configurable labels; no Hungarian notation or `I` prefix on TS interfaces.
- **2026-05-20** — **Backend reusability**: domain logic in `app/Modules/<Engine>/Domain/` is framework-agnostic (no Eloquent, no HTTP); Application services (one public method per use case) orchestrate domain + persistence + events; Repositories own all Eloquent queries; HTTP controllers ≤ 30 lines (validate → call service → return resource); cross-module reusable code lives in `app/Support/`; DRY at domain level, WET at boundaries.
- **2026-05-20** — **Frontend reusability**: components in `src/components/ui/` (pure, presentational); `src/components/<feature>/` (feature-bound containers); `src/components/site/` (shell pieces); hooks own data-fetching + state shape; API client in `src/lib/api/` (generated from OpenAPI); no business calculations in UI; server components by default.
- **2026-05-20** — **API versioning & deprecation**: all endpoints live under `/api/v1/`; breaking changes go to `/api/v2/` (never silently changed); within a major version, changes must be additive only; old endpoints marked with `Deprecation` and `Sunset` headers and kept alive ≥ 90 days after replacement ships; resource URLs plural nouns (`/api/v1/matches`); filtering via query params; standard envelope + error shape; error codes are stable strings (`engine.specific_reason`).
- **2026-05-20** — **Database indexing discipline**: index every foreign key (mandatory); index every column in WHERE/ORDER BY/JOIN on hot paths; composite indexes put most-selective column first, equality before range; partial indexes for mostly-null columns or filtered queries (e.g. `WHERE archived_at IS NULL`); time-series tables (analytics, audit logs) partitioned monthly; EXPLAIN required on hot-path reads before merge; unused indexes dropped after 30 days in production.
- **2026-05-20** — **Pagination strategy**: cursor-based (`?cursor=<opaque>&limit=<n>`) on hot endpoints (feeds, ledgers) for stability; offset pagination only for admin views; hot list endpoints expose `etag` + honor `If-None-Match`.
- **2026-05-20** — **Testing pyramid**: many unit tests (deterministic, no clocks/network/filesystem), fewer integration tests (full stack including DB + outbox + listener), few e2e tests; 80% line coverage threshold on domain code enforced in CI; test names describe behavior (`it("locks stake when both parties confirm the challenge")`), not naming pattern; contract tests validate event payloads and API responses against schemas in `contracts/`.
- **2026-05-20** — **Performance SLOs**: public read paths target p95 < 200ms cached / < 500ms uncached; write paths target p95 < 400ms end-to-end including outbox write; every hot read is cached (Cloudflare edge for public pages, Redis for application-level); cache keys namespaced; invalidation is explicit via `cache.invalidate` event; no synchronous outbound HTTP in request lifecycle (use queue/job with retries); bulk operations chunk data.
- **2026-05-20** — **Error handling & observability**: structured JSON logging (timestamp, level, request_id, event, context); never log secrets, OTPs, evidence URLs, or unhashed phone numbers; every request has a `request_id` propagated through logs/events/downstream calls; OpenTelemetry traces every HTTP request and queue job; metrics emitted at every boundary (HTTP latency, queue depth, outbox lag, cache hit rate, DB pool saturation); dashboards owned per-engine; alerts on SLOs not raw metrics.
- **2026-05-20** — **Git & PR discipline**: branch names `wp/<wp-id>-<slug>` (e.g. `wp/WP-20260520-stood-ground-cost`); commits are imperative present tense ("Add stood-ground cost config key"); one concern per commit when feasible; PRs stay small (soft limit 400-line diff excluding generated code); never `git push --force` to shared branch without heads-up; never bypass pre-commit/pre-push hooks (`--no-verify`).
- **2026-05-20** — **Security layered on OWASP Top 10**: mass assignment blocked (no implicit `$fillable`), explicit DTOs; CSRF on stateful endpoints; rate limiting (strict on auth + OTP endpoints); inputs validated by both FormRequest (PHP) and Zod (TS); R2 evidence bucket private, signed URLs only (5-min TTL), audit-logged; signed URLs never in logs; secrets via `.env` + Vault, never committed; dependencies kept current (Dependabot + Renovate, critical CVEs ≤ 7 days); SQL injection impossible (parameterized queries); XSS impossible on Next (no `dangerouslySetInnerHTML` with user input, sanitize with `DOMPurify` if needed).
- **2026-05-20** — **Money is integer minor units (pesewas), never floats.** Booking commission, settlement, refunds — all integer pesewas, converted to display currency only at API boundary.
- **2026-05-20** — **Events & outbox versioning**: event names follow `engine.action_past_tense` (`match.result_confirmed`); events carry canonical envelope (event_id UUIDv7, schema_version, occurred_at, actor_id, actor_role, source, payload); producers write to outbox in same DB transaction as domain write; outbox relay publishes to Redis Streams asynchronously; consumers are idempotent (dedupe by event_id + listener_name); schema version bumped when payload fields added/removed/retyped, not for additive-only changes.
- **2026-05-20** — **Configuration discipline**: every configurable value registered in `contracts/config/<engine>/<key>.yaml` (key, engine, scope, value_type, default, approval_level, description); code reads via `Config::get(key, scope, at?)`, never raw `env()` or `process.env` outside entry points; no magic numbers in domain code (5, 30, 0.05, 100 are all config keys in disguise); effective-dated reads so historical values are reconstructable.
- **2026-05-20** — **Documentation discipline**: code comments on _why_, not what; no commented-out code; README per engine in `docs/engines/<slug>/` is canonical entry point; ADRs in `docs/adr/` for non-obvious architectural decisions; glossary updated with new domain terms; `docs/JOURNAL.md` owned by Sankofa (off-limits to manual edit except reorganization).

## Glossary

- **Sankofa** — The project's continuous documentation subagent; named after the Akan symbol meaning "go back and fetch it." Owns and maintains `docs/JOURNAL.md`.
- **Respect Points (RP)** — The non-monetary currency of the Challenge system; transferred between clubs based on call-out outcomes (win, draw, forfeit, ducked).
- **Fan Buzz** — A normalized hype score for a Challenge, mixing reactions, WhatsApp shares, predictions, comments, and view velocity. Displayed as a percentile against rolling-average buzz, not an absolute number.
- **Scene pack** — The hand-curated library of Challenge card background art plates (gauntlet, lightning, war-drums, etc.) the card composer draws from.
- **Work Packet (WP)** — Atomic unit of work that flows through all 10 Agent OS pipeline stages. Identified by `WP-YYYYMMDD-<slug>`. Every PR carries the WP ID and must show all 10 stage boxes ticked before merge. Example: `WP-20260520-configurable-draw-cost`.
- **Agent OS** — The governance meta-layer (10-stage pipeline, Constitution, chat modes, contracts, ADR cadence) that sits on top of Build Plan and describes _how_ every change is built (intake → release packet), orthogonal to _what & when_ (Build Plan's phasing).
